## Chapter 01 자바스크립트 개요와 개발환경 설정

책에서 사용할 프로그램이나 개발환경에 대한 내용은 이미 알고 있기에 넘어갔다. Chrome과 VSCode는 이미 사용중이므로 다운 받을 필요가 없었다.

자바 스크립트의 기초적인 문법의 내용 또한 다른 언어에서 많이 다룬 부분이기에 가볍게 훑어보며 차이점만 확인했다. 가장 큰 차이점은 **'와 "의 차이 아닐까?** 생각한다. 변수 선언 부분이나 다른 부분들도 자바와 매우 유사하므로 크게 어려움이 없었다.

챕터 1에서는 코드를 작성한 부분의 확인을 위한 출력 부분을 유의 깊게 봤다. Chrome의 개발자 도구 console 부분에서 바로 간단한 표현식을 사용할 수 있는 부분은 상당히 마음에 든다. 자바는 간단한 표현식도ㄴ 하나의 클래스에 main 메스드를 만들고 print 메서드를 사용해야하는 과정이 있어서 불편했지만 이런 부분이 없는 것은 매우 만족스럽다.

```javascript
alert('Hello World')
console.log('Hello World')
```

간단한 내용은 alert, 많은 내용과 복잡한 출력은 console.log를 사용하면 될 것 같다. 2개의 함수를 암기하는 것으로 이 챕터를 마무리했다.

## Chapter 02 자료와 변수


### 1 기본 자료형

- 템플릿 문자열

백틱(`)을 사용하는 방식으로 문자열 안의 숫자와 같은 변수를 편하게 입력하게 해줌
```javascript
let num = 10
alert(`${number}은 2 자리 숫자입니다.`)
>> 출력 10은 2 자리 숫자입니다.
```


- 다른 자료형과의 비교

자바에 비해 자바 스크립트는 바꿔서 비교해서 같게 만들어줌
```javascript
1 == '1'
>> 자료형을 변환 후 같은 1이 되므로 같음
false == "0"
>> false가 0으로 "0"이 0으로 변환되므로 같음
"" == []
>> 빈 문자열은 false, 빈 배열도 false로 변환되어 같음
0 == []
>> 0은 false, 빈 배열도 false로 변환되어 같음
```

### 2 상수와 변수

자바에서 상수는 final로 잘 사용하지 않는데 자바 스크립트에서는 자주 사용하는 것 같음
변수는 자바와 거의 동일하게 사용 됨
각 선언은 변수 상수를 정의하는 것으로 정의하는 부분이 다름.
자바는 어떤 자료형인지가 중요하다면 자바 스크립트는 변수/상수를 조금 더 중시하는 것 같음.

선언 하는 방식 

> 상수 - const 
변수 - let(var)

기타 연산 방식은 동일

자바에는 없는 undefined 자료형이 존재하는데 정의하지 않고 선언 시 undefined 자료형을 가짐
자바는 자료형 없이 선언이 불가능 함.

상당히 느슨한 규칙을 지닌 언어 같음

### 3 자료형 변환

10*"10* = 100 으로 자바스크립트가 내부에서 자료형을 변환 해줌

사용자로부터 문자열 입력은 prompt("입력창에 보이는 메시지", "기본 입력 값")
불 입력(예/아니오)는 confirm("입력창에 보이는 메시지")로 입력받음

자바에 비해 각 자료형 변환은 굉장히 느슨함

문자열은 +를 사용하면 뒤에 대부분 다 변환해줌 boolean, null, 심지어 undefined도 붙음
숫자는 (- * / %)를 사용하면 변환 가능함
Boolean이 조금 특이한데 5가지 제외하면 모두 true임.(0, NaN, "", '', null)

메서드를 이용한 방식

```javascript
Number()  - 숫자
String()  - 문자열
Boolean() - 불린
```

## Chapter 04 반복문

### 1 배열

#### 배열 선언

자바와 다르게 배열에 동일한 자료형을 넣지 않아도 된다.
```javascript
const arr = [3, 'str', [3, "str"], true, function() {}, {}]
```

#### 배열 추가

자바와 다르게 배열에 값을 추가하는데 제약이 없는 수준이다.

```javascript
arr = [1, 2]

arr.push(3)
>> (3) [1, 2, 3]

arr[10] = 4
>> (11) [1, 2, 3, empty × 7, 4]

```

push와 같은 함수를 이용할 수도 있고 그냥 원하는 인덱스에 값을 넣을 수 있다.
늘어나게 되면 빈 인덱스는 empty로 자동으로 채워진다.


#### 배열 삭제

배열 삭제는 2가지로 나뉜다.
- 인덱스 기반
- 값을 기반

**인덱스로 제거하는 함수**

`배열.splice(인덱스, 제거할 요소의 개수)`

```javascript

arr = [1, 2, 3, 4, 5]

arr.splice(2,2)
>> (2) [3, 4]  // 제거된 요소를 보여줌
arr
>> (3) [1, 2, 5]

```

**값을 기반으로 삭제**

값으로 요소를 제거하려면 특정 값을 찾는 `indexOf()`함수를 이용하여 인덱스 값을 추출하여 삭제한다.

```javascript
const 인덱스 = 배열.inedxOf(찾고자 하는 값)
배열.splice(인덱스, 1)
```

찾고자 하는 값의 인덱스를 추출하여 `splice()` 함수를 이용하여 해당 인덱스로부터 1개만 삭제한다.

```javascript
const arr = ['사과', '바나나', '배']
> undefined
const idx = arr.indexOf('바나나')
> undefined
idx
> 1
arr.splice(idx, 1)
> ['바나나']
arr
> (2) ['사과', '배']
```

#### 배열의 특정 위치에 요소 추가하기

자바에서는 불가능한 기능으로 편리하다.
`splice()`함수를 이용하여 넣고자 하는 인덱스, 0, 넣고자 하는 요소의 3개의 값을 넣어서 사용한다.

`배열.splice(인덱스, 0, 요소)`

```javascript
const arr = ['사과', '바나나', '배']
> undefined
arr.splice(1, 0, "복숭아")
> []length: 0[[Prototype]]: Array(0)
arr
> (4) ['사과', '복숭아', '바나나', '배']
arr[1]
> '복숭아' // 내가 넣고자한 1번 인덱스에 넣고자 하는 요소인 복숭아가 들어감
```


기억해야할 점은 해당 인덱스 뒤에 넣는 것이 아닌 해당 인덱스 앞에 요소가 들어간다.
한마디로 내가 입력한 인덱스에 요소가 생성되는 것이다.

### 2 반복문

for, while의 기본적인 사용법은 자바와 똑같다.
하지만 자바 스크립트에는 for in과 for of가 존재하기에 이 부분만 유심히 봤다.

#### for in

배열의 인덱스를 이용하는 방식이다.

```javascript
for (const 반복 변수 in 배열 또는 객체) {
  문장
}
```

```javascript
const todo = ['빨래', '청소', '산책']
> undefined

for (const i in todo) {
    console.log(`${i}번재 할 일은 ${todo[i]}입니다.`)
}
> 0번재 할 일은 빨래입니다.
> 1번재 할 일은 청소입니다.
> 2번재 할 일은 산책입니다.
```

처음에는 인덱스 i가 값을 벗어나는 것을 신경쓰지 않고 사용해도 되서 굉장히 좋다고 생각했다. 하지만 책에서 안정적인 사용을 위해서는 i의 다음 값이 존재하는 지 확인하는 if문을 같이 써야 안정적으로 사용할 수 있다고 하는 것처럼 만능은 아니다. 자주 사용하는 함수는 아니라고 하며 기억만 하고 지나가야 할 것 같다.

#### for of

```javascript
for (const 반복 변수 of 배열 또는 객체) {
  문장
}
```

for in 과 다르게 '문장'에서 반복 변수가 들어간다.

```javascript
const todo = ['빨래', '청소', '산책']
> undefined

for (const td of todo) {
    console.log(`오늘 할 일은 ${td}입니다.`)
}
> 오늘 할 일은 빨래입니다.
> 오늘 할 일은 청소입니다.
> 오늘 할 일은 산책입니다.
```

자바의 향상된 for문의 `for(변수 : 배열)`과 매우 유사하게 사용하면 될 것 같다.
